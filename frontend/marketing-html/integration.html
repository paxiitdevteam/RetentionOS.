How to Integrate RetentionOS

Simple 5-minute setup. One script tag. Start saving revenue today.

How It Works

RetentionOS uses a hybrid model: Sign up like Salesforce/Jira, integrate like Stripe/Intercom.

1️⃣

Sign Up & Get API Key

Create your RetentionOS account (like Salesforce or Jira). Get your API key from the dashboard settings. Takes 2 minutes.

2️⃣

Add Widget Script

Add one JavaScript snippet to your platform (like Stripe or Intercom). The widget automatically detects cancel button clicks.

3️⃣

Configure Flows

Design retention flows in the dashboard (like Salesforce). Set up pause, downgrade, discount, and support steps.

5-Minute Quick Start

Get RetentionOS running on your platform in minutes.

Step 1: Get Your API Key

After signing up, go to Settings → API Keys in your dashboard. Click "Generate New API Key" and copy it.

Step 2: Add Widget Script

Add this code before the closing </body> tag on pages where users can cancel: 

<script src="https://cdn.retentionos.com/widget.js"></script>
<script>
  RetentionOS.init({
    apiKey: 'ros_live_your_api_key_here',
    userId: '{{current_user_id}}',
    plan: '{{user_plan}}',
    region: '{{user_region}}'
  });
</script>

Step 3: Replace Placeholders

  • ros_live_your_api_key_here → Your API key from dashboard
  • {{current_user_id}} → Current user's ID in your system
  • {{user_plan}} → User's subscription plan (e.g., 'pro', 'basic')
  • {{user_region}} → User's region code (e.g., 'us', 'eu')

Step 4: Create Your First Flow

Go to Flows in your dashboard. Create a new flow with steps like "Pause Subscription", "Downgrade to Basic", or "20% Discount". The widget will automatically use this flow when users click cancel.

✅ That's it! The widget will automatically detect cancel button clicks and show your retention flow.

  document.body.appendChild(script);
}, [user]);

How It Works (Technical Flow)

From cancel click to saved revenue in 8 steps.

1

User Clicks Cancel

User clicks "Cancel Subscription" button on your platform.

2

Widget Detects Click

RetentionOS widget automatically detects the cancel button click.

3

Widget Calls API

Widget sends request to RetentionOS API with user info.

4

Backend Returns Flow

RetentionOS returns the appropriate retention flow (pause, downgrade, discount).

5

Widget Shows Modal

Beautiful modal displays retention options to the user.

6

User Makes Decision

User accepts offer (pause/downgrade/discount) or continues cancellation.

7

Backend Processes

If accepted, RetentionOS updates Stripe subscription and logs event.

8

Analytics Update

Dashboard shows saved revenue, saved users, and churn insights.
